# Silver Research Bot — 后续完善多维度分析报告

> 生成日期: 2026-06-15 | 基于 6 轮升级后的项目状态

---

## 上下文

项目已完成 6 轮大规模升级（RAG 混合检索、记忆系统、论文搜索、Pipeline 增强、Agent 基础设施、前端交互），当前处于功能完备但测试/安全/部署/可观测性薄弱的状态。以下从 10 个维度分析后续改进方向。

---

## 一、测试体系深化

| 方向 | 方案 |
|---|---|
| **Mock LLM Provider** | 创建 MockProvider 返回固定响应，使所有 Pipeline 阶段可离线测试 |
| **API 集成测试** | FastAPI TestClient + 标准 PDF fixture → 端到端 |
| **RAG 精度评估** | ground truth 数据集 → MRR/NDCG@k |
| **翻译质量回归** | 标准段落 + 参考译文 → BLEU/COMET 自动评分 |
| **公式检测基准** | 标注 PDF 公式位置 → precision/recall/F1 |
| **快照测试** | Pipeline 各阶段输出 snapshot，prompt 变更自动对比 |

**关键缺口**: 零 mock 能力——任何测试都依赖真实 LLM API，无法在 CI 中运行。

---

## 二、部署与运维

| 方向 | 方案 |
|---|---|
| **Docker 化** | Dockerfile + docker-compose.yml (backend + frontend) |
| **CI/CD** | GitHub Actions: lint → test → build → push |
| **进程守护** | systemd / supervisord / pm2 管理 uvicorn |
| **健康检查增强** | Provider 连通性、磁盘空间、RAG 索引状态 |
| **数据库迁移** | 论文数 > 10K 时 index.json → SQLite/PostgreSQL |
| **配置热加载** | 修改 config.json 不重启生效 |

**关键缺口**: 无容器化——无法一键部署。

---

## 三、安全加固

| 方向 | 方案 |
|---|---|
| **API 认证** | JWT / API Key 中间件，多用户隔离 workspace |
| **速率限制** | slowapi 或自实现令牌桶 |
| **PDF 安全检查** | 文件大小限制、PDF 结构校验、恶意 JS 检测 |
| **LLM 输出过滤** | HTML sanitize（Bleach），iframe srcdoc XSS 防护 |
| **CORS 锁紧** | `allow_origins=["*"]` → 白名单域名 |
| **审计日志** | 记录所有 API 调用的 user/timestamp/action/ip |

**关键缺口**: `allow_origins=["*"]` + 零认证 → 公网部署即可被滥用。

---

## 四、可观测性

| 方向 | 方案 |
|---|---|
| **结构化日志** | JSON 格式 + trace_id + paper_id + stage + duration_ms |
| **LLM 成本追踪** | model/tokens/cost 按 paper/session 聚合 |
| **Pipeline 延迟分析** | 各阶段耗时分布 → 识别瓶颈 |
| **错误监控** | Sentry/webhook 推送 Pipeline 失败/超时/磁盘不足 |
| **Dashboard** | 实时：正在分析论文数/日均调用次数/月成本/错误率 |

**关键缺口**: 完全不知道每篇论文花了多少 token（成本）。

---

## 五、RAG 系统进阶

| 方向 | 方案 |
|---|---|
| **GraphRAG** | 实体-关系图（方法→数据集→指标→作者），多跳推理 |
| **查询改写** | LLM 改写为多角度检索查询 → 结果融合 |
| **HyDE** | LLM 生成假设答案 → 用假设答案做向量检索 |
| **对话式 RAG** | 多轮对话跟踪检索上下文，自动消歧指代 |
| **引用溯源** | 检索结果标注原文出处（第几页第几段） |
| **RAG 评估闭环** | 用户反馈（有用/无用）→ 自动调整检索权重 |

**关键缺口**: 当前 RAG 不支持对话历史上下文——每次查询独立不携带上一轮信息。

---

## 六、Pipeline 深化

| 方向 | 方案 |
|---|---|
| **方法论谱系** | 提取"使用了谁的方法"→构建全局方法演进树 |
| **数据集卡片** | 自动生成 Dataset Card（来源/规模/标注/偏差） |
| **数学推导验证** | LLM 逐步检查论文中关键推导的正确性 |
| **代码仓库搜索** | 从论文标题/作者搜索 GitHub 官方实现 |
| **多语言支持** | 当前仅 en/zh → 扩展到日/韩/法/德文 |
| **PPT 生成** | 分析结果自动生成 presentation slides |

**关键缺口**: AUDIT 发现问题后只标记不修复——缺少 self-correction loop。

---

## 七、Agent 能力扩展

| 方向 | 方案 |
|---|---|
| **MCP 生态接入** | 社区 MCP servers（GitHub/ArXiv/Playwright/Python REPL） |
| **Agent 技能市场** | 用户可分享/安装自定义技能包 |
| **多模态 Agent** | 图片输入（论文截图→OCR→分析） |
| **长期任务管理** | Agent 多天研究任务，checkpoint/恢复 |
| **主动建议** | 监测 ArXiv 新论文→自动推送推荐 |
| **工具学习** | Agent 从失败的工具调用中学习优化参数 |

**关键缺口**: Agent 无法处理图片（论文中的图表、公式截图）。

---

## 八、前端体验

| 方向 | 方案 |
|---|---|
| **响应式布局** | 底部 TabBar + 汉堡菜单（当前仅桌面可用） |
| **暗/亮主题切换** | 浅色模式 CSS 变量 |
| **PDF 内嵌阅读器** | PDF.js 双栏对照（原文/译文同步滚动） |
| **公式交互** | 点击公式弹出解释卡片 |
| **图表画廊** | 提取图表网格展示，点击放大 |
| **全局搜索** | 跨论文标题/摘要/公式搜索 |
| **键盘快捷键** | Ctrl+K 搜索, Ctrl+Enter 发送 |
| **通知系统** | 分析完成浏览器推送通知 |

**关键缺口**: 无响应式——手机上基本不可用。

---

## 九、数据与分析

| 方向 | 方案 |
|---|---|
| **阅读历史** | 分析过的论文按时间线展示，笔记/书签 |
| **个性化推荐** | 基于阅读历史+标签 → 推荐相关论文 |
| **研究趋势** | 聚合关键词/方法 → 领域热度趋势图 |
| **协作功能** | 多人共享论文分析结果，评论/讨论 |
| **导出格式** | 除 ZIP 外支持 Notion/Obsidian 格式 |

## 十、架构演进

| 方向 | 方案 |
|---|---|
| **插件系统** | Pipeline Stage 可插拔注册 |
| **消息队列** | RabbitMQ/Redis 解耦上传与分析 |
| **向量数据库** | ChromaDB/Milvus 替代 numpy |
| **微服务拆分** | Extract/Analyze/RAG 独立服务 |
| **多租户** | 用户隔离 workspace + quota 管理 |
| **API 版本化** | `/api/v1/...` / `/api/v2/...` |

---

## 总结：改进优先级矩阵

| 优先级 | 维度 | 改进项 | 理由 |
|---|---|---|---|
| **P0** | 安全 | API 认证 + 速率限制 + CORS | 部署前硬前提 |
| **P0** | 测试 | Mock LLM Provider | 无 mock 无法 CI |
| **P1** | 部署 | Docker 化 + docker-compose | 一键部署 |
| **P1** | 可观测 | LLM 成本追踪 | 运营必需 |
| **P1** | Pipeline | Audit 修复 loop | 质量闭环 |
| **P1** | RAG | 对话式 RAG | 体验跃升 |
| **P2** | 前端 | 响应式 + PDF 阅读器 | 移动端 + 核心体验 |
| **P2** | Agent | MCP 社区生态 | 工具扩展 |
| **P3** | GraphRAG | 实体-关系图检索 | 工程量大天花板高 |
| **P3** | 架构 | 插件系统 + 向量数据库 | 长期演进 |
