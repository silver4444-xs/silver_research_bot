## 一、严重问题

### 1. `loop.py:359` — `self._config` 未定义导致 AgentLoop 初始化崩溃

**文件**: `silver_research_bot/agent/loop.py` 第 359 行

```python
paper_cfg = getattr(self._config, "paper_search", None)
```

`self._config` 在整个 `__init__` 中从未被赋值。该行在 `_register_default_tools()` 中被调用，该函数在构造函数中执行，导致每次实例化 `AgentLoop` 时抛出 `AttributeError`，Agent 循环完全无法启动。

**影响**: Agent 对话功能不可用（如果有使用 AgentLoop 的路径）。

**修复**: 添加 `config` 构造参数并从 `Config.load()` 传入，或移除该代码并用 `ToolsConfig` 字段替代。

### 2. RAG 全量重索引导致向量翻倍（数据损坏 Bug）

**文件**: `silver_research_bot/research_rag.py` 第 330-342 行

```python
async def reindex(self) -> dict[str, Any]:
    self._ensure_init()
    self._rebuild_bm25()
    if self._embedder and self._vector_store:
        self._vector_store = type(self._vector_store)(self.root / "vectors", dim=self._embedding_dim)
```

新 `VectorStore` 构造函数立即从磁盘 `_load()` 旧向量，随后 `add()` 追加所有 chunks，导致每次重索引向量数翻倍。重复调用使搜索质量持续退化、内存/磁盘占用不断膨胀。

**影响**: RAG 搜索质量随时间衰减。

**修复**: 构造新 store 前删除旧 `.npy`/`.pkl` 文件，或使用纯内存空 store。

### 3. RAG / Research 全栈零并发保护（9 个文件）

**影响范围**: `research_rag.py`, `research_embedder.py`, `research_vector_store.py`, `research_bm25.py`, `research_core.py`, `research_app.py`（全局变量段）, `paper_analyzer/manager.py`

代码库其他部分大量使用 `asyncio.Lock` + `asyncio.Semaphore`（`agent/memory.py`, `agent/loop.py`, `api/server.py` 等），但 Research/RAG 全栈 9 个文件没有任何并发原语。

具体竞态窗口:

| 位置 | 问题 |
|------|------|
| `research_rag.py:74-88` | `_ensure_init()` TOCTOU — 两个并发请求同时通过守卫，重复初始化 |
| `paper_analyzer/manager.py:36-39` | `_save_index()` — 并发 `register_paper`/`update_status`/`delete_paper` 互相覆盖索引 |
| `research_embedder.py:28,56,82` | `_cache` 为普通 `dict` — 并发访问不安全 |
| `research_vector_store.py:43-48` | `_save()` 三文件非原子写入 — 崩溃时状态不一致 |
| `research_app.py:79-97` | `_get_rag()` 全局懒加载 — 并发初始化创建多个实例 |

**修复**: 在所有共享可变状态访问处添加 `asyncio.Lock`。

### 4. 前端移动端导航完全不可用

**文件**: `web/src/style.css` 第 592 行

```css
@media (max-width: 768px) {
    .sidebar { display: none; }
```

768px 以下时整个侧边栏被 `display:none` 隐藏，无汉堡菜单、无底部标签栏、无抽屉导航。移动端用户无法切换任何标签页（论文/Agent/RAG/历史/趋势）。应用在移动端功能上完全不可用。

**修复**: 添加底部标签栏或汉堡菜单作为移动端导航方案。

---

### 5. PDF 全量渲染消耗 GPU 内存

**文件**: `web/src/App.vue` 第 383 行

`renderAllPdfPages()` 从第 1 页渲染到最后一页，为每一页创建独立 canvas。200 页 PDF = 200 个 canvas，消耗数百 MB GPU 内存。无虚拟滚动、无按需渲染、无页面回收。

**修复**: 实现 IntersectionObserver 驱动的虚拟滚动，仅渲染可见页 ± 1 缓冲区。

## 二、高优先级

### 6. 配置加载错误静默回退默认值

**文件**: `silver_research_bot/config/loader.py` 第 55-58 行

```python
except (json.JSONDecodeError, ValueError, pydantic.ValidationError) as e:
    logger.warning(f"Failed to load config from {path}: {e}")
    logger.warning("Using default configuration.")
```

任何配置错误（JSON 语法错、字段类型错、必需字段缺失）都静默回退全默认值。输错 API Key 字段名导致无 LLM 调用但仍正常启动。

**修复**: 区分"文件不存在"（回退默认值合理）与"文件存在但格式错误"（应中止启动并输出清晰错误）。

### 7. MCP 服务器允许任意命令执行（安全漏洞）

**文件**: `silver_research_bot/config/schema.py` 第 273-291 行

```python
class MCPServerConfig(BaseModel):
    command: str = ""
    args: list[str] = Field(default_factory=list)
    env: dict[str, str] = Field(default_factory=dict)
```

`MCPServerConfig` 接受任意 `command` + `args` + `env` 无任何验证。`ChannelsConfig` 的 `extra="allow"` 进一步扩大攻击面。

**修复**: 添加命令白名单、限制工作目录为沙箱路径、对 `args` 做参数注入防护。

### 10. API 输入无验证

**文件**: `silver_research_bot/research_app.py`

- `paper_id` 无格式验证（应限制为 UUID/hex 正则防路径遍历）
- `UploadFile` 无大小限制（可能磁盘耗尽）
- 无请求速率限制
- CORS `allow_origins=["*"]`

**修复**: 添加 `paper_id` 格式验证、文件大小上限（50MB）、`slowapi` 限流。

---

### 11. Service Worker 无 `activate` 事件

**文件**: `web/sw.js`

- 无 `activate` 事件 → 旧缓存永不清理，`silver-research-v1` 永久占用存储
- 无 `clients.claim()` → 新 SW 不控制已打开的标签页
- `cacheFirst` 策略 → 部署后用户看到旧版本直到手动清除
- `install` 预缓存仅包含 `/` 和 `/index.html`，缺少 Vite 构建产物

**修复**: 添加 `activate` 事件清理旧缓存 + `clients.claim()` + `stale-while-revalidate` 策略。

---

### 12. CDN 安全风险

**文件**: `web/index.html`

5 个 CDN 资源（MathJax 3, Mermaid 10, PDF.js 3.11, D3 v7, Google Fonts）:
- 无 Subresource Integrity (SRI) 哈希 → 供应链攻击面
- 版本不一致: MathJax `@3`（大版本浮动）/ PDF.js `@3.11.174`（精确版本）
- 无 fallback → CDN 故障时公式/图表/PDF 全部失效
- Google Fonts 阻塞首次渲染
- 总大小 ~3.2MB 未压缩，全量 eager 加载

**修复**: 添加 SRI 哈希 + 本地 fallback + 非关键库懒加载。

---

### 13. MCP 连接存在竞态条件

**文件**: `silver_research_bot/agent/loop.py` 第 370-390 行

```python
if self._mcp_connected or self._mcp_connecting or not self._mcp_servers:
    return
self._mcp_connecting = True
```

无锁的 check-then-act — 两个协程同时通过守卫导致重复连接和工具注册。

**修复**: 使用 `asyncio.Lock` 保护 `_connect_mcp`。

### 15. 前端代码组织

**文件**: `web/src/App.vue` (413 行单文件)

- **50+ 函数** 全部全局作用域: Markdown 渲染器、LaTeX 清理器、PDF 渲染器、审计渲染器、D3 图表
- **40+ reactive refs** 扁平命名空间: `tab`, `ps`, `dt`, `hs`, `upMode`, `papers`, `pdet`, `cmpIds`, `chatMsgs`, ...
- **无 Vue Router** → 所有页面切换通过 ref 值硬编码
- **无 Pinia** → 无状态管理，跨组件通信靠全局 ref
- **无 TypeScript** → API 响应形状无类型保护

**性能问题**:
- `renderMd()` 每次击键重新正则解析全文
- `sanitizeLatex()` 每次渲染执行 15+ 次顺序 `.replace()`
- `MathJax.typesetPromise()` 全局重新渲染所有公式而非增量
- 无 `v-memo` / computed 缓存渲染结果

### 18. BM25 性能退化（O(N·D)）

**文件**: `silver_research_bot/research_bm25.py` 第 83 行

```python
doc_entry = next((d for d in self._docs if d["id"] == doc_id), None)
```

内层循环线性扫描 `self._docs`，嵌套在唯一查询词 × 匹配文档的三层循环中。10000 chunks 时每次搜索 ~1 亿次迭代。
