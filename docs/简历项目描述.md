# SILVER RESEARCH BOT — 论文深度研读 AI Agent 框架

## 1.

**技术栈**: Python, FastAPI, LangChain, asyncio, PyMuPDF, Vue 3, D3.js, MathJax, Mermaid, PostgreSQL+pgvector, OpenAI/Anthropic/DeepSeek 等 30+ LLM Provider

**项目描述**: 基于 nanoBot Agent 框架构建的论文深度研读 AI 系统，实现从 PDF 上传到 8 阶段全自动分析的端到端管线，涵盖翻译、多维分析、公式解读、可视化、引用图谱与审稿审计，帮助研究人员高效理解与评估学术论文。

- 基于 asyncio 编排 8 阶段顺序分析管线（PDF 解析→翻译→四维并行分析→公式批量解读→程序化可视化→引用图谱→A/B 三视角审稿→质量审计），每阶段产出独立 Markdown/HTML/JSON 文档，支持 WebSocket 实时进度推送。
- 基于 PyMuPDF 实现 PDF 深度解析，包含原图提取（doc.extract_image）、80+ Unicode→LaTeX 符号转换、公式 5 规则过滤（运算符+变量/带参 LaTeX/希腊+其他/数字+运算符/已知函数名），拒绝孤立符号与 URL/DOI 误检。
- 设计公式提取与解读全链路：翻译路径 $...$→独立行 $$...$$ 自动升级→_is_complete_formula 二次过滤→LLM 四级解读（符号定义/数学含义/领域含义/关联关系），结合 MathJax 前端渲染与 .fmean 行内 LaTeX 后处理。
- 基于 LLM 构建引用图谱提取管道，从全文 16000 字中自动识别论文/基础/对比/背景四类引用关系，通过 D3.js v7 力导向图实现交互式引用网络可视化。
- 设计三视角 A/B 审稿机制（理论家/工程派/领域专家），并行生成独立审稿意见，结合结构完整性检查与 LLM 深度审计实现自动化论文质量评估。
- 基于 Vue 3 + Vite 构建单文件 SPA 前端（~1500 行），集成 PDF.js v3.11 双栏同步阅读器、D3.js 可视化仪表板、MathJax 公式交互弹窗、i18n 中英双语切换、深色科技风 CSS 主题。

## 2.

**技术栈**: Python, FastAPI, asyncio, BM25, numpy, ChromaDB/pgvector, LLM Cross-Encoder, LangChain

**项目描述**: 面向学术文献场景的混合 RAG 检索系统，实现关键词与语义的多路召回融合，结合 LLM 重排序与多模态过滤，为文献检索与论文研读提供高精度上下文支持。

- 设计三阶段混合检索架构：BM25 Okapi 关键词粗排 top-20 + Embedding 向量相似度召回 top-20 并行执行，经 min-max 归一化后加权融合（0.3×BM25 + 0.7×Vector），再由 LLM Cross-Encoder 重排序输出 top-5。
- 构建多模态过滤层，支持 text/formula/figure/table 四类内容分通道检索，基于 JSONL append-only 存储 + numpy 向量索引实现高效增量入库与全量重建。
- 实现文献全生命周期管理（CRUD + tombstone 软删除 + reindex），结合研究趋势分析（10 领域关键词匹配 + D3.js 柱状/热力图/折线图三图联动）。

## 3.

**技术栈**: Python, asyncio, FastAPI, LangChain, PostgreSQL, pgvector, Langfuse, MCP 协议

**项目描述**: 面向本地可审计场景的 AI Agent 工作台，围绕 Agent 执行过程的可观测性、安全性与长期记忆展开设计，实现对对话、记忆检索、工具调用及 Prompt 注入的全链路追踪与审计。

- 设计基于「对话→记忆检索→工具选择→响应生成」的 Agent 执行链路模型，明确各阶段输入输出结构，实现可复现的推理路径记录与 Langfuse 全链路追踪。
- 设计三层记忆架构（Active Memory 逐轮注入 → Project Memory 跨 session 共享 → Long-term Memory 持久归档），结合 LLM 1-10 重要性评分、语义冲突检测（contradiction/duplicate/update）、Ebbinghaus 遗忘曲线（R=e^(-t/S)，半衰期 7 天）实现智能记忆管理。
- 基于 Token 预算驱动的 Consolidator 实现对话历史自动压缩，通过 LLM 摘要 + Dense Retrieval + BM25 + RRF 融合排序降低约 80% 无效上下文 Token 开销。
- 抽象 Multi-Agent 协作团队机制，支持 Translator + Analyzer + Auditor 三 Agent 通过 MessageBus 异步协同，各 Agent 绑定专属 tools + temperature + SOUL.md 角色定义。
- 设计可扩展 LLM Provider 层（30+ 提供商统一接口），实现智能重试（1/2/4s 指数退避 + 429 分类限流/配额区分）、图片自动降级、角色交替修正等容错机制。
