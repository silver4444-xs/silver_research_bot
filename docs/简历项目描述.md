# SILVER RESEARCH BOT — 论文深度研读 AI Agent 框架

## 1.

**技术栈**: Python, FastAPI, asyncio, PyMuPDF, Vue 3, D3.js, MathJax, Mermaid, NumPy, OpenAI/Anthropic/DeepSeek 等 29 LLM Provider

**项目描述**: 独立设计并实现的论文深度研读 AI Agent 系统，从 PDF 上传到 8 阶段全自动分析端到端闭环，涵盖翻译、四维并行分析、公式解读、可视化、引用图谱与审稿审计，将单篇论文研读时间从"天"压缩到"分钟"。底层自研 Agent 框架（ReAct 循环 + 艾宾浩斯记忆 + 14 工具 + Skill/MCP 体系），无 LangChain 依赖。

- **8 阶段 Pipeline 编排**：基于 asyncio 的顺序+并行混合管线（PDF 解析→LaTeX 保护翻译→四维 `asyncio.gather` 并行分析→公式批量 LLM 解读→Mermaid 程序化可视化→D3.js 引用图谱→三视角并行审稿→质量审计），每阶段独立产出，WebSocket 实时进度推送。
- **公式提取全链路**：PyMuPDF 深度解析（原图 xref 提取 + 80+ Unicode→LaTeX 映射）→ 5 规则过滤（运算符/变量/带参 LaTeX/希腊符号/函数名）→ 碎片合并与假阳性清洗。经历 4 轮迭代优化，召回率从 70% 提升至 99.7%，建立 17 个 Eval 用例守卫质量。
- **公式解读与渲染**：翻译路径 `$...$` → `$$...$$` 自动升级 → 补全过滤 → LLM 四级解读（符号定义/数学含义/领域语境/关联关系），结合 MathJax 前端渲染与点击弹窗交互，支持 `.fmean` 行内 LaTeX 后处理与 `&` 转义加固。
- **引用图谱与审稿审计**：LLM 从全文中自动识别论文/基础/对比/背景四类引用 → D3.js v7 力导向交互网络。三视角并行审稿（理论家/工程派/领域专家）+ 结构完整性检查 + LLM 深度审计（严重/一般/建议三级分级）。
- **Vue 3 单文件 SPA**（~1500 行）：PDF.js v3.11 双栏同步阅读器、D3.js 三图联动（柱状+热力+折线）、MathJax 公式交互弹窗、i18n 中英切换、深色科技风主题（CSS 变量 + Grid + Glow 特效，~800 行），全部 CDN 加载零前端构建依赖。

## 2.

**技术栈**: Python, FastAPI, asyncio, BM25, NumPy, LLM Cross-Encoder

**项目描述**: 面向学术文献场景的混合 RAG 检索系统，自研实现关键词与语义多路召回融合，零外部向量数据库依赖（纯 NumPy 文件存储），结合 LLM 重排序与多模态过滤，为文献检索与论文研读提供高精度上下文支持。

- **三阶段混合检索架构**：BM25 Okapi 关键词粗排 top-20 + Embedding 向量相似度召回 top-20 `asyncio.gather` 并行执行，经 min-max 归一化后加权融合（0.3×BM25 + 0.7×Vector），再由 LLM Cross-Encoder 重排序输出 top-5。相比单一检索方式显著提升召准率。
- **多模态过滤层**：支持 text/formula/figure/table 四类内容分通道检索，基于 JSONL append-only 存储 + NumPy 向量索引实现高效增量入库与全量重建索引。
- **文献全生命周期管理**：CRUD + tombstone 软删除 + reindex 全量重建，结合研究趋势分析（10 领域关键词匹配 + D3.js 柱状/热力/折线三图联动）。

## 3.

**技术栈**: Python, asyncio, FastAPI, NumPy, MCP 协议, Langfuse（可选）

**项目描述**: 自研 AI Agent 工作台，无 LangChain 依赖。围绕 Agent 执行过程的可观测性、安全性与长期记忆展开设计，从零实现 ReAct 循环引擎（1,487 行）、三层记忆架构、14 工具体系与 Skill/MCP 扩展机制。

- **Agent 核心循环**：从零实现 ReAct（Thought→Action→Observation）循环引擎，含中轮消息注入（`asyncio.Queue`）、崩溃检查点恢复、上下文五步治理（孤立结果清理→缺失回填→微压缩→预算→裁剪）、重复调用拦截、死循环防护（max_iterations + 空重试上限 + 注入上限），可选 Langfuse 全链路追踪。
- **三层记忆架构**：Active Memory（逐轮检索注入）→ Project Memory（跨 session 共享）→ Long-term Memory（持久归档），结合 LLM 1-10 重要性评分、语义冲突检测（contradiction/duplicate/update）、Ebbinghaus 遗忘曲线（`R=e^(-t/S)`，半衰期 7 天）实现智能记忆管理。设计 Dream 两阶段后台整合——LLM 分析历史交互 → AgentRunner 编辑持久化记忆文件。
- **上下文治理**：Token 预算驱动的 Consolidator 自动压缩对话历史，微压缩将旧工具结果替换为 `[omitted]` 摘要，`_enforce_role_alternation()` 确保消息角色合规，固定 System Prompt 前缀利用 LLM prompt cache 降低 50%+ token 消耗。
- **Multi-Agent 协作**：PaperAnalysisTeam（Translator + Analyzer + Auditor），三 Agent 通过 MessageBus（`asyncio.Queue` 双队列异步生产者-消费者模式）协同，各 Agent 绑定专属 tools + temperature + SOUL.md 角色定义。
- **29 LLM Provider 统一层**：ABC 抽象基类 + OpenAI Compat 适配层（90% Provider 共用）+ Anthropic 专用适配。智能重试（1/2/4s 指数退避 + 429 分类限流/配额区分）、base64 图片自动降级、角色交替自动修正。
